# Memory Seek 登录界面设计文档

## 概述

为 Memory Seek 前端应用设计一个基础登录界面，使用 Rust + WASM 技术栈（Leptos 框架），支持深色/浅色模式切换。

## 技术栈

- **前端框架**: Leptos (Rust WASM)
- **构建工具**: Trunk
- **路由**: leptos_router
- **共享类型**: memory-seek-type（与后端共享的请求/响应模型）

## 设计规格

### 布局

采用**居中卡片式**布局，登录表单在页面中央显示。

### 视觉风格

- **整体风格**: 素雅简洁，无多余装饰
- **浅色模式**:
  - 背景色: 象牙白 `#FFFFF0`
  - 卡片背景: 纯白 `#FFFFFF`
  - 边框: 暖色 `#f0ede4`
  - 按钮: 深灰 `#4a4a4a`
  - 文字: 深灰 `#2c2c2c`，辅助文字 `#8a8a8a`
- **深色模式**:
  - 背景色: `#1a1a1a`
  - 卡片背景: `#2a2a2a`
  - 边框: `#3a3a3a`
  - 按钮: 象牙白 `#e8e4d9`
  - 文字: `#f0ede4`，辅助文字 `#777`

### 功能模块

#### 1. 登录功能

**账号密码登录**（当前后端仅支持此方式）
- 输入项: 账号（用户名/邮箱）、密码
- 调用 API: `POST /auth/login`
- 请求参数: `{ account, password }`

> 注：邮箱验证码登录需要后端新增支持，当前版本暂不实现。

#### 2. 注册功能

通过外层 Tab 切换到注册页面，包含以下字段：
- 用户名
- 邮箱
- 密码
- 昵称
- 邀请码
- 邮箱验证码

调用 API:
1. `POST /auth/verification-codes` 发送邮箱验证码
2. `POST /auth/register` 提交注册

#### 3. 忘记密码

> 注：当前后端未提供忘记密码/重置密码的 API，此功能暂不实现。可在 UI 上预留入口但显示"功能开发中"提示。

### 交互流程

1. **登录成功后的行为**:
   - 如果有上一页（被重定向到登录页的），返回上一页
   - 否则跳转到首页

2. **Token 存储**:
   - 使用 `localStorage` 持久保存
   - 存储 `accessToken` 和 `refreshToken`

3. **主题切换**:
   - 支持深色/浅色模式
   - 可通过系统偏好或手动切换

### 与后端 API 对接

基于现有后端 API：

| 功能 | 方法 | 路径 | 请求体 |
|------|------|------|--------|
| 登录 | POST | /auth/login | `{ account, password }` |
| 注册 | POST | /auth/register | `{ username, email, password, nickname, inviterCode, emailVerifyCode }` |
| 发送验证码 | POST | /auth/verification-codes | `{ email }` |
| 刷新 Token | POST | /auth/token | Header: `x-user-id`, `x-refresh-token` |

> 注：当前后端仅支持账号密码登录，邮箱验证码登录和忘记密码功能需要后端新增 API 支持。

## 验收标准

1. 登录页面居中卡片式布局正确显示
2. 浅色模式（象牙白）和深色模式都能正常工作
3. 登录/注册 Tab 切换流畅
4. 账号密码登录功能正常
5. 表单验证（前端）正常工作
6. 登录成功后正确跳转（有上一页返回上一页，否则跳首页）
7. Token 正确存储到 localStorage
8. 注册流程完整（包含邮箱验证码发送）
9. 忘记密码入口显示"功能开发中"提示
