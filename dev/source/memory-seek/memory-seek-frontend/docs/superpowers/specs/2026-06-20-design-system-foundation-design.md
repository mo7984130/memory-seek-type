## 7. 动画系统

### 7.1 过渡速度

| 名称 | 速度 | 用途 |
|------|------|------|
| 快速 | 150ms | 按钮、开关、Tooltip |
| 常规 | 200ms | 下拉菜单、标签切换 |
| 平滑 | 300ms | 模态框、页面切换 |

### 7.2 缓动函数

| 名称 | 函数 | 用途 |
|------|------|------|
| ease-out | cubic-bezier(0, 0, 0.2, 1) | 进入动画、展开、显示 |
| ease-in-out | cubic-bezier(0.4, 0, 0.2, 1) | 过渡、位移、切换 |
| spring | cubic-bezier(0.34, 1.56, 0.64, 1) | 强调、反馈、庆祝 |

### 7.3 双端支持

#### 鼠标端 (Desktop)

- 使用 `@media (hover: hover)` 检测
- 支持 hover 悬停效果
- 动画效果：translateY、boxShadow

#### 触屏端 (Mobile)

- 使用 `:active` 伪类
- 支持按压反馈效果
- 动画效果：scale、opacity
- 使用 `@media (pointer: coarse)` 增大触摸区域

### 7.4 场景应用

| 场景 | 鼠标端 | 触屏端 |
|------|--------|--------|
| 按钮 | hover: translateY(-2px) | active: scale(0.95) |
| 卡片 | hover: translateY(-4px) | active: scale(0.98) |
| 输入框 | focus: border + shadow | focus: border + shadow |

## 8. 布局系统

### 8.1 栅格系统

- **类型：** 12 列栅格
- **特点：** 经典方案，灵活度高，适合复杂布局

### 8.2 容器宽度

- **最大宽度：** 1200px
- **类型：** 响应式容器，根据屏幕宽度自适应
- **内边距：** 16px（移动端）/ 24px（桌面端）

### 8.3 响应式断点

| 名称 | 宽度 | 用途 |
|------|------|------|
| mobile | 768px 以下 | 移动端布局 |
| desktop | 769px 以上 | 桌面端布局 |

### 8.4 页面布局

#### 桌面端

- 顶部导航栏（固定）
- 侧边栏（可选）
- 主内容区（12 列栅格）

#### 移动端

- 顶部导航栏（固定）
- 主内容区（2 列栅格）
- 底部导航栏（固定）

## 10. 设计原则

### 10.1 简约

- 界面简洁，减少视觉干扰
- 内容优先，突出核心功能
- 留白充足，呼吸感强

### 10.2 一致

- 统一的设计语言
- 一致的交互模式
- 可预测的用户体验

### 10.3 现代

- 现代化的视觉风格
- 流畅的动画效果
- 响应式设计

### 10.4 可访问性

- 足够的颜色对比度
- 合理的触摸区域
- 清晰的视觉反馈
